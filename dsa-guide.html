Understand DSA patterns in plain English

You write C here — loops, arrays, pointers, structs. The patterns below are ways of thinking, not magic syntax. Read one section, then open the matching question in DSA Patterns and use Visualizer ⧉.

DFS — Depth-First Search

One sentence: Go as deep as you can, then back up and try another branch — like exploring a maze by always taking the first turn until you hit a dead end.

Grid (1 = land)     DFS from (0,0):
1 1 1 0 0           Visit → sink to 0 (flood fill)
1 1 0 0 0           Same connected component = 1 island
0 0 0 1 1
0 0 0 0 1

In C you usually write

  • A function that calls itself on neighbors: dfs(g, r+1, c), dfs(g, r-1, c), …
  • A base case: out of bounds, or already visited, or wrong cell value → return.
  • Mark visited (e.g. flip '1' to '0') so you don't loop forever.

Practice: Number of Islands (q118) · Tree Path Sum (q120)

BFS — Breadth-First Search

One sentence: Explore in layers — first all cells 1 step away, then 2 steps, then 3 — like ripples in a pond. Best for shortest path on an unweighted grid.

Start (0,0)  dist=1
  layer 1: (1,0) (0,1)  dist=2
  layer 2: (2,0) (1,1) (0,2)  dist=3
  … first time you reach the goal = minimum steps

In C you usually write

  • A queue array: store (row, col, distance), with head and tail indices.
  • A vis[][] array so you enqueue each cell once.
  • Pop front, try 4 neighbors; if open and unvisited, push back with dist+1.

Practice: Shortest Path Grid (q119)

Dynamic programming (DP)

One sentence: Break a big problem into small overlapping sub-problems; store answers in a table so you never recompute the same thing twice.

Climbing 5 stairs (1 or 2 steps at a time):
dp[0]=1  dp[1]=1
dp[2]=dp[1]+dp[0]=2
dp[3]=dp[2]+dp[1]=3
dp[4]=5  dp[5]=8  ← answer

How to recognize DP

  • “How many ways…”, “minimum cost…”, “maximum without adjacent…” → often DP.
  • Define dp[i] in plain English first (e.g. “best loot using houses 0..i”).
  • In C, you can use a small array or just two variables if only the last two rows matter.

Practice: Climbing Stairs (q111) → House Robber (q112) → Coin Change (q113)

Greedy

One sentence: At each step, take the locally best choice and never look back — no table needed if the problem is designed for it.

Jump Game — index:  0  1  2  3  4
nums[i]:           2  3  1  1  4
farthest you can:  2  4  4  4  8  → reach index 4 ✓

In C you usually write

  • One loop, a few variables (e.g. farthest, end of interval).
  • Update the variable when you see a better local option.

Practice: Jump Game (q116) · Non-Overlapping Intervals (q117) · Merge Intervals (q122)

Monotonic stack

One sentence: Keep a stack of indices whose values are in sorted order; when you see a “breaking” value, pop and answer questions about the popped items.

Temps: 73 74 75 71 69 72 76 73
Stack holds decreasing temps → when today is warmer,
pop yesterday and set wait = today - yesterday

Practice: Daily Temperatures (q114) · Next Greater Element (q115)

Suggested learning order

  1. Jump Game — one array, greedy
  2. Climbing Stairs — simplest DP
  3. Number of Islands — grid DFS
  4. Shortest Path Grid — grid BFS
  5. House Robber — DP choice
  6. Daily Temperatures — stack
  7. Then coin change, intervals, tree path sum
