DFS — Depth-First Search

One sentence: Go as deep as possible along one path, then backtrack and try the next branch — like walking a maze and always taking the first open turn until you hit a wall.

Real-life analogy

You're exploring a building floor plan with a pen. Enter a room, mark it visited, open the first door, keep going until every reachable room from that entrance is marked. That's one "connected component." Start again from any unmarked room for the next component.

Interactive trace — flood fill on a grid

In Number of Islands, each group of adjacent '1' cells is one island. DFS sinks each cell to '0' so you never visit twice.

Step 1 / ?

Minimal C skeleton

/* Grid DFS — 4 directions */
void dfs(char g[][COLS], int rows, int cols, int r, int c) {
    /* 1. BASE CASE — stop if out of bounds or not land */
    if (r < 0 || r >= rows || c < 0 || c >= cols) return;
    if (g[r][c] != '1') return;

    /* 2. MARK VISITED — sink so we don't revisit */
    g[r][c] = '0';

    /* 3. RECURSE — try all 4 neighbors */
    dfs(g, rows, cols, r + 1, c);
    dfs(g, rows, cols, r - 1, c);
    dfs(g, rows, cols, r, c + 1);
    dfs(g, rows, cols, r, c - 1);
}

Line-by-line trace (first call dfs(g, 4, 5, 0, 0))

  1. Enter (0,0): in bounds, g[0][0]=='1' → set to '0'
  2. Recurse down: dfs(1,0) → same island, mark and continue
  3. Recurse right: from (0,0) also try (0,1), (0,2) … entire blob gets sunk
  4. Return: when all four recursive calls finish, this cell's DFS is done
  5. Outer loop (in main's helper): scan grid; next '1' at (3,3) starts a new island count
Use DFS when: counting connected regions, exploring all paths, tree recursion (left/right), or mutating the grid in place.
Tree example: Tree Path Sum (q120) — recurse left and right with a smaller target.

BFS — Breadth-First Search

One sentence: Explore in expanding rings — all cells 1 step away first, then 2 steps, then 3. The first time you reach the goal, you have the shortest path (on an unweighted grid).

Real-life analogy

Ripples in a pond. Drop a stone at the start — the wave hits nearby cells first. BFS uses a queue (FIFO: first in, first out) so cells are processed in order of distance.

Interactive trace — shortest path on a grid

Open cells expand layer by layer. A wall forces the path to go around. When the goal cell is dequeued, its distance is the answer.

Step 1 / ?

Minimal C skeleton

/* Queue stores (row, col, dist). head/tail indices. */
int q_r[MAX], q_c[MAX], q_d[MAX];
int head = 0, tail = 0;

void enqueue(int r, int c, int d) {
    q_r[tail] = r; q_c[tail] = c; q_d[tail] = d; tail++;
}

/* In shortest-path helper: */
int vis[ROWS][COLS] = {0};
enqueue(0, 0, 1); vis[0][0] = 1;

while (head < tail) {
    int r = q_r[head], c = q_c[head], d = q_d[head]; head++;
    if (r == goal_r && c == goal_c) return d;   /* found shortest */
    /* for each open neighbor: if (!vis[nr][nc]) { vis=1; enqueue(nr,nc,d+1); } */
}

Line-by-line trace

  1. Enqueue (0,0) dist=1 — queue is not empty, so BFS can run
  2. Dequeue front — process the oldest cell first (that's what makes it "breadth" first)
  3. Push neighbors with dist+1 only if unvisited — mark in vis[][]
  4. Goal check when dequeuing — not when enqueueing — so distance is correct
Use BFS when: shortest steps on a grid, level-order traversal, or "minimum number of moves."

DFS vs BFS — quick comparison

DFS

  • Stack (implicit via recursion) or explicit stack
  • Goes deep first
  • Great for: flood fill, all paths, trees
  • Does not guarantee shortest path

BFS

  • Queue (head / tail)
  • Expands layer by layer
  • Great for: shortest path on unweighted grid
  • Uses more memory at wide layers
Tree DFS

Trees — same DFS idea, different shape

One sentence: A binary tree is nodes with left and right pointers. DFS on a tree = recurse into children instead of grid neighbors.

Grid DFS visits 4 neighbors. Tree DFS usually visits 2 children. The base case is root == NULL. Leaf = both children are NULL.

Interactive trace — path sum to 22

Subtract each node value from the target as you go down. At a leaf, check if remaining is 0.

Step 1 / ?

Minimal C skeleton

struct Node { int val; struct Node *left, *right; };

int hasPathSum(struct Node *root, int target) {
    if (!root) return 0;
    if (!root->left && !root->right)
        return root->val == target;   /* leaf */
    return hasPathSum(root->left, target - root->val)
        || hasPathSum(root->right, target - root->val);
}

BST insert/search/traverse live in the Binary Search Tree category (q63–q70). Tree path sum is the bridge from grid DFS to pointer recursion.

Table / recurrence

Dynamic programming (DP)

One sentence: Solve small versions of the problem, save answers in a table, and build up to the big answer — so you never solve the same sub-problem twice.

How to think (the part that confused you)

DP is not a special library or keyword in C. It's a recipe:

  1. Define dp[i] in English: "number of ways to reach stair i" or "best loot using houses 0..i"
  2. Find the recurrence: how does dp[i] relate to smaller indices?
  3. Base cases: dp[0], dp[1] you know by hand
  4. Fill the table left to right (or nested loops for 2-D)

Interactive trace — Climbing Stairs (n=5)

Same problem, three implementations. Switch mode, then step through each.

Top-down + memo

Step 1 / ?

Minimal C — two variables instead of a full array

int climbStairs(int n) {
    if (n <= 1) return 1;
    int a = 1, b = 1;   /* dp[0], dp[1] */
    for (int i = 2; i <= n; i++) {
        int c = a + b;    /* dp[i] = dp[i-1] + dp[i-2] */
        a = b;
        b = c;            /* slide the window */
    }
    return b;
}

Memoization vs tabulation (both are DP):
Top-down — recursive function + cache array: "if dp[i] already computed, return it." Feels like DFS with memory.
Bottom-up — fill dp[0..n] in a loop (what q111–q113 use). Same math, often easier in C interviews.
Start with tabulation; switch to memo if the recurrence is naturally recursive (e.g. tree DP).
One pass

Greedy

One sentence: At each step, make the locally best choice and never reconsider — often just one loop and one or two variables. No table needed.

Interactive trace — Jump Game

From index i you can jump up to nums[i] steps. Track farthest — the rightmost index reachable so far.

Step 1 / ?

Minimal C skeleton

int canJump(int *nums, int n) {
    int farthest = 0;
    for (int i = 0; i < n && i <= farthest; i++) {
        if (i + nums[i] > farthest)
            farthest = i + nums[i];
    }
    return farthest >= n - 1;
}

Interval problems (q117, q122) use the same spirit: sort by end time, keep the greedy choice, one pass.

Stack

Monotonic stack

One sentence: Walk the array once; keep a stack of indices whose values stay in sorted order. When the current value "breaks" the order, pop and answer questions about what you popped.

Diagram — Daily Temperatures

Stack Indices with decreasing temps. Day 3 (71) is warmer than stack top → pop day 2, wait[2]=1

Minimal C skeleton

int stack[MAX], top = -1;

for (int i = 0; i < n; i++) {
    while (top >= 0 && T[i] > T[stack[top]]) {
        int prev = stack[top--];
        wait[prev] = i - prev;   /* warmer day found */
    }
    stack[++top] = i;
}

Search

Backtracking

One sentence: Try a choice, explore deeper, then undo the choice and try the next — like walking a decision tree where you back up at dead ends.

How it differs from DFS

Grid DFS marks cells visited permanently (sink to '0'). Backtracking un-does each pick so another branch can reuse the same slot. Classic for subsets, permutations, N-queens, Sudoku.

Template in C

void backtrack(int start) {
    record_answer();                    /* current partial solution counts */
    for (int i = start; i < n; i++) {
        choose(i);                      /* pick nums[i] */
        backtrack(i + 1);                /* explore with this pick */
        unchoose(i);                    /* often implicit when i loops */
    }
}

Trace — subsets of {1,2,3}

  1. Call bt(0): count {} → try pick 1
  2. Call bt(1): count {1} → try pick 2
  3. Call bt(2): count {1,2} → try pick 3 → count {1,2,3}
  4. Unwind: also get {1,3}, {2,3}, {2}, {3}, … → 8 total
